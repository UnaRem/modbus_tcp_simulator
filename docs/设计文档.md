# 设计文档：Linux 高级 Modbus TCP 设备仿真系统

**版本：** 1.0  
**来源：** `docs/需求规格说明书.md` 第 14 章“实现细化”拆分  

---

## 1. 设计目标

- 将需求转化为可直接实现的模块与接口。
- 明确并发模型、数据结构、请求处理流程与异常处理策略。
- 给出脚本执行与物理引擎的实现约束，保证确定性与稳定性。

---

## 2. 模块边界与职责

- **配置加载与校验**: 读取 YAML，完成 Schema 校验、默认值填充、错误聚合返回。
- **虚拟总线控制器**: 维护 `Listener -> SlaveID -> DeviceContext` 路由表，完成请求分发。
- **设备模型**: 保存寄存器定义与当前值，提供读写、范围校验、权限校验。
- **物理引擎**: 定时执行仿真与跨设备联动逻辑。
- **脚本运行器**: 提供安全 API，管理脚本执行周期与超时。
- **监控与日志**: 采集指标、输出结构化日志。

---

## 3. 并发与线程模型

- **监听器隔离**: 每个 TCP 端口使用独立的监听任务，互不阻塞。
- **物理引擎独立**: 使用单独的调度线程或任务周期执行，避免阻塞 Modbus 请求。
- **脚本执行策略**: 默认串行执行脚本，确保确定性；如需并行必须保证脚本无共享写冲突。

---

## 4. 请求处理流程

- **步骤 1**: 解析请求中的 `Unit ID`，查找目标设备上下文。
- **步骤 2**: 校验功能码与地址范围，失败返回异常码。
- **步骤 3**: 对写入操作进行权限与范围校验，失败返回异常码。
- **步骤 4**: 成功读写后更新内部寄存器值并记录日志与指标。

## 8. 校验规则（实现级）

- **CFG-ERR-01**: 同一 `listener` 下 `slave_id` 重复 -> 配置错误。
- **CFG-ERR-02**: 同一设备内寄存器地址重复 -> 配置错误。
- **RT-ERR-01**: 读写地址越界 -> 异常码 `0x02`。
- **RT-ERR-02**: 写入值超范围 -> 异常码 `0x03`。
- **RT-ERR-03**: 写只读寄存器 -> 异常码 `0x03`。

---

## 9. 物理引擎时间步进

- **时间源**: 使用单调时钟计算 `dt`，避免系统时间回拨。
- **步长控制**: `dt` 超过 `interval * 2` 时截断为 `interval * 2`，避免异常跳变。
- **数值边界**: SoC、电压等状态在写入前执行边界保护。

## 12. 异常处理规范（模块级）

- `validate_config` 仅收集错误，不抛异常，方便一次性提示。
- `handle_request` 任何异常都需转换为 Modbus 异常响应。
- 脚本超时与异常必须被捕获并记录，且不影响主循环。

---

## 13. 数据编码细节（实现级）

- `int16/uint16` 直接映射到单个寄存器。
- `int32/uint32/float32` 使用两个寄存器，高字在前（可配置）。
- 脚本 API 读写时执行缩放：外部值 = 内部值 * scale，写入时反向换算并做范围校验。
- Modbus 请求读写使用原始寄存器值，不做缩放换算。
- `string` 使用 `length` 指定寄存器数量，每个寄存器按 2 字节拼接。
- `bits` 仅作为元数据，不改变寄存器读写方式。

## 17. 线程/并发与性能模型

### 17.1 线程划分

- 主线程负责启动与关闭所有组件。
- 每个监听端口单独线程运行 Modbus Server。
- 物理引擎单独线程运行，按固定周期调用 `tick`。

### 17.2 数据一致性与锁策略

- 每个 `DeviceContext` 维护独立 `lock`。
- 读写操作进入 `RegisterStore` 前必须获取锁。
- 物理引擎与 Modbus 请求共享同一锁策略，避免竞态。

### 17.3 性能与背压

- 连接上限通过 `runtime.max_connections` 控制。
- QPS 限制通过令牌桶实现，超限返回异常码 `0x06`。
- 单请求处理时间超过 `runtime.request_timeout_ms` 记录告警。

### 17.4 性能采样规则

- 延迟采样以 P95 为准，按 60 秒滚动窗口统计。
- QPS 统计按 1 秒窗口计算。

## 19. 脚本执行沙箱与超时策略

### 19.1 沙箱边界

- 禁止导入任何模块，不提供 `__import__`。
- 仅暴露白名单 API：`get_value`、`set_value`、`get_meta`、`log`。
- 限制脚本长度与变量数量，默认最大 64KB。

### 19.2 执行模式

- 脚本在物理引擎线程内按周期执行。
- 脚本在启动时编译为代码对象缓存。

### 19.3 超时策略

- 单次执行超过 `timeout_ms` 记录告警并标记脚本为 `disabled`。
- 被禁用脚本不再参与后续调度，需手动恢复。

### 19.4 异常处理

- 任何脚本异常被捕获并记录，不影响主循环。
- 连续 3 次异常自动禁用脚本。

## 21. 接口级行为约束（Behavior Contracts）

### 21.1 配置加载与校验

- `ConfigLoader.load` 仅负责 I/O 与 YAML 解析，失败抛出 `ConfigError`。
- `ConfigNormalizer.apply_defaults` 在校验前执行，填充默认值与派生字段。
- `ConfigValidator.validate` 返回错误列表，不抛异常。

### 21.2 RegisterStore 读写语义

- `read(address, count)` 返回原始寄存器值列表，`count <= 0` 视为非法请求。
- `write(address, values)` 仅对可写寄存器生效，触发权限与范围校验。
- 地址超出已定义范围返回异常码 `0x02`。
- 写入越界或只读寄存器返回异常码 `0x03`。

### 21.3 Script API 行为

- `get_value` 返回工程值（应用 `scale`）。
- `set_value` 接收工程值，内部反向换算为原始寄存器值并校验。
- 访问不存在寄存器抛出 `ScriptError` 并计入脚本错误。

## 22. Modbus 报文与地址规则

### 22.1 地址基准

- 使用 `listeners[].address_base` 控制地址基准，允许 0 或 1。
- 解析请求时执行：`real_addr = req_addr + address_base`。

### 22.2 最大读写长度

- `0x03`/`0x04` 最大读取 125 个寄存器。
- `0x10` 最大写入 123 个寄存器。
- 超限请求返回异常码 `0x03`。

### 22.3 异常码优先级

- 功能码非法优先返回 `0x01`。
- 地址非法优先返回 `0x02`。
- 权限或数值非法返回 `0x03`。
- 运行时过载或限流返回 `0x06`。

---

## 23. 生命周期与资源回收

- `start` 完成配置加载、监听器启动、物理引擎启动。
- `stop` 顺序为：停止物理引擎 -> 停止监听器 -> 刷新日志与指标。
- 任何异常退出必须记录 `fatal` 级日志并返回非零退出码。
